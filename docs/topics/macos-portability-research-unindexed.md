# macOS Portability Research

Estado: estacionado e indexado como referencia profunda. No cambia el roadmap Windows-first.

Objetivo: guardar lo aprendido sobre un posible port a macOS sin convertirlo todavia en roadmap ni decision.

## Resumen

Portar Copicu a macOS parece viable si se separa en capas:

1. core portable: UI, SQLite, busqueda, scripts, blobs, settings y copy-back;
2. integracion macOS basica: menu bar, shortcut global, autostart, bundle;
3. integracion nativa dificil: detectar app anterior, restaurar foco y pegar automaticamente;
4. clipboard rich macOS: mapear `NSPasteboard`/UTType hacia el modelo MIME-first.

El riesgo real no es Tauri sino el flujo tipo CopyQ: activar item, escribir clipboard, volver al target previo y mandar paste.

## Lo Que Apps Reales Hacen

### Maccy

Maccy es un clipboard manager macOS open source, keyboard-first y minimalista.

Hallazgos utiles:

- Hotkey por defecto: `Shift+Command+C`.
- Enter selecciona/copia; `Option+Enter` elige item y pega.
- Tiene opcion de pegar automaticamente, pero su FAQ indica que requiere agregar Maccy a Privacy & Security -> Accessibility.
- Monitorea el pasteboard con intervalo configurable. El README documenta default de 500 ms y `clipboardCheckInterval` para bajarlo.
- Tiene lista de tipos de pasteboard a ignorar para datos transitorios/confidenciales: `org.nspasteboard.TransientType`, `org.nspasteboard.ConcealedType`, `org.nspasteboard.AutoGeneratedType`, ademas de tipos de apps como 1Password.
- Recomienda inspeccionar tipos con Pasteboard-Viewer para crear reglas de ignore.

Lectura para Copicu:

- En macOS, polling de `NSPasteboard.changeCount` cada 500 ms es un pattern aceptado por apps reales.
- El primer port puede copiar al clipboard sin permiso especial, pero paste automatico debe depender de Accessibility.
- Conviene tener ignore-types nativos para no capturar datos temporales o secretos.

Fuentes:

- https://github.com/p0deje/Maccy
- https://maccy.app/

### Clipy

Clipy es otro clipboard manager open source para macOS.

Hallazgos utiles desde el codigo:

- `ClipService.startMonitoring()` usa un timer Rx cada 500 ms, lee `NSPasteboard.general.changeCount`, compara contra cache local y crea un clip cuando cambia.
- Al guardar, filtra tipos disponibles, apps excluidas y apps especiales antes de persistir.
- `PasteService.paste()` escribe el item en `NSPasteboard.general`, verifica Accessibility, y luego genera `Command+V` con `CGEvent`.
- Antes de pastear, incrementa su contador local en algunos caminos para evitar recapturar el item que acaba de escribir.
- El README advierte que macOS asocia permisos de Accessibility con la firma de codigo; builds sin firma estable pueden pedir permiso repetidamente.

Lectura para Copicu:

- El equivalente macOS de nuestro self-write suppression puede apoyarse en hash + contador local, pero hay que validar contra `changeCount`.
- La ruta de paste concreta para macOS probablemente sea: write item to `NSPasteboard` -> hide picker/menu -> verify Accessibility -> post `Cmd+V` via `CGEvent`.
- La firma/notarization no es solo release polish: afecta estabilidad de permisos de Accessibility durante dogfood.

Fuentes:

- https://github.com/Clipy/Clipy
- https://github.com/Clipy/Clipy/blob/develop/Clipy/Sources/Services/ClipService.swift
- https://github.com/Clipy/Clipy/blob/develop/Clipy/Sources/Services/PasteService.swift

### CopyQ

CopyQ ya documenta el modelo funcional que Copicu imito en Windows.

Hallazgos utiles:

- Return copia el item completo al clipboard; si "Paste to current window" esta habilitado, envia un shortcut de paste a la ventana previa.
- El shortcut default documentado es `Shift+Insert`, aunque se puede cambiar a `Ctrl+V`.
- CopyQ documenta que paste funciona en Windows, macOS y X11, pero no en Wayland.
- Para fallos de paste, sus causas comunes son foco incorrecto o formato no soportado por target.
- Hay issue macOS donde Enter dejo de pegar por perder Accessibility; el workaround fue quitar/reagregar CopyQ en Privacy -> Accessibility.

Lectura para Copicu:

- El contrato host actual sigue siendo correcto: `writeItem` primero, luego `focusPrevious`, luego `sendPasteShortcut`.
- macOS debe tener failure policy explicita para permiso de Accessibility, foco y app target.
- Linux/Wayland debe tratarse separadamente; no conviene extrapolar desde macOS.

Fuentes:

- https://copyq.readthedocs.io/en/latest/faq.html
- https://github.com/hluk/CopyQ/issues/1030

### Raycast Clipboard History

Raycast no es open source, pero su manual confirma expectativas de producto macOS:

- primary action configurable: paste to active app o copy to clipboard;
- opcion de paste as plain text;
- disabled applications para no registrar Passwords, Keychain Access, bancos, password managers;
- OCR local de imagenes para hacer capturas buscables.

Lectura para Copicu:

- Para Mac, "copy" y "paste" deben ser acciones separadas y configurables.
- Privacy controls por app no son nice-to-have; son parte del baseline de clipboard manager serio.
- OCR de imagenes puede quedar futuro, pero el modelo de metadata/search debe permitirlo.

Fuente:

- https://manual.raycast.com/clipboard-history

### Apple

- `NSPasteboard`/`NSPasteboardItem`: clipboard/pasteboard item model.
- `NSPasteboard.changeCount`: practical signal for polling.
- Uniform Type Identifiers (`UTType`): mapear `public.utf8-plain-text`, `public.html`, `public.png`, archivos, etc.
- `NSWorkspace.frontmostApplication`: detectar app activa/anterior.
- `AXIsProcessTrustedWithOptions`: verificar/pedir Accessibility.
- `CGEvent`: sintetizar `Command+V` u otros shortcuts.

## Corte Recomendado Si Se Retoma

### Fase 1: compilar y correr sin paste automatico

- Hacer que `windows` crate y `WebView2Loader.dll` sean Windows-only.
- Agregar bundle macOS en Tauri sin romper NSIS Windows.
- Cambiar textos `Windows startup` a texto neutral.
- Validar main picker, settings, SQLite, busqueda, scripts, blobs y copy-back texto.
- Registrar shortcut global con default macOS.
- Menu bar app basica.

Resultado esperado: Copicu funciona como clipboard history + copy-back en Mac.

### Fase 2: clipboard monitor macOS

- Spike `NSPasteboard.changeCount` polling 500 ms.
- Capturar texto plain.
- Capturar tipos ignorables/transient/concealed y no persistirlos.
- Validar self-write suppression despues de `writeItem`.
- Validar imagen screenshot-to-clipboard, browser image, Preview, Finder file copy.

Resultado esperado: captura confiable sin paste automatico.

### Fase 3: paste automatico macOS

- Guardar app anterior con `NSWorkspace.frontmostApplication` antes de abrir picker.
- Al activar item: escribir `NSPasteboard`, ocultar picker, reactivar app anterior si hace falta, verificar Accessibility, postear `Cmd+V`.
- Mostrar estado/CTA si falta Accessibility.
- Tests manuales con Safari/Chrome/VS Code/Terminal/Finder/TextEdit.

Resultado esperado: equivalente macOS de `focusPrevious + sendPasteShortcut`.

### Fase 4: packaging/distribucion

- Firmar app para que Accessibility no se resetee en cada build distribuido.
- Notarizar si se va a distribuir fuera de dev.
- Documentar permisos y failure modes.

## Preguntas Abiertas

- Para Mac, `Enter` deberia copiar y `Option+Enter` pegar como Maccy, o conservar `Enter` copy y `Shift+Enter` paste como Copicu Windows actual?
- Usar polling `NSPasteboard.changeCount` propio o confiar primero en `clipboard-rs` y solo agregar fallback macOS si falla?
- Que default de shortcut Mac evita conflictos: `Cmd+Shift+C`, `Cmd+Shift+V`, `Cmd+Shift+,` u otro?
- El port Mac debe empezar con una app menu bar sin ventana visible al launch, igual que Windows tray?
- Cual es el minimo de signing necesario para dogfood sin pelear contra permisos de Accessibility?
