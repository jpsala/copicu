## Riesgos Que No Hay Que Olvidar

- Paste-to-previous-window es el flujo mas riesgoso.
- Clipboard rich content puede explotar el alcance.
- No persistir datos reales del clipboard en docs, tests o logs.
- No perseguir paridad CopyQ antes de estabilizar el MVP.
- En Windows/Tauri, tocar APIs de UI/ventana/plugin desde callbacks nativos o threads propios puede dejar la app `Responding=False` o perceptualmente unresponsive; usar callbacks rapidos + `app.run_on_main_thread(...)`.
- En global shortcuts compuestos, no emitir eventos backend hacia el WebView principal ni registrar bare next-step global shortcuts sin harness dedicado; puede romper IPC renderer->Tauri aunque React/CDP sigan vivos. Preferir que el renderer consulte estado pending y capture el siguiente paso enfocado.
- En ventanas secundarias Tauri/WebView2, no asumir que se comportan como `main`. Antes de culpar React, clasificar por separado: existencia HWND/list_apps, posicion/tamano, CDP/renderer heartbeats, IPC `record_renderer_diagnostic`, DOM/texto accesible, y captura visual. Un renderer puede recibir estado (`whichkey-sync pending=...`) y aun asi la HWND no componer contenido.
