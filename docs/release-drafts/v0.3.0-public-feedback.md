## Summary

This release refreshes Copicu's public onboarding and adds a clearer script/action showcase for Windows power users. It is still a Windows alpha: the app is used daily by the maintainer, but APIs, script contracts, packaging, and some native behaviors can still change.

## Highlights

- Repositioned Copicu as a local-first, scriptable clipboard manager for Windows power users.
- Reworked README to front-load install, demo, core flows, privacy, limitations, and contribution paths.
- Added clearer public launch docs and drafts for technical/community feedback.
- Added more realistic synthetic demo data for screenshots, docs, and future recordings.
- Added a README-ready synthetic picker screenshot and refreshed the generated picker demo GIF/MP4/poster.
- Added two trusted local script examples:
  - clean URL tracking parameters and copy the cleaned URL;
  - format selected JSON and copy the pretty-printed result.
- Added a focused test for URL tracking cleanup behavior.
- Improved script registry empty-state onboarding in Settings.
- Added code-level good-first-issue candidates for contributors.

## What To Test

Please use synthetic or non-sensitive clipboard content only when reporting issues.

Most useful feedback:

- clipboard capture from specific Windows apps;
- paste-to-previous-window behavior in Notepad, browsers, VS Code/editors, terminals, chat apps, and admin/elevated windows;
- picker search, keyboard navigation, preview readability, and focus behavior;
- local script/action workflows that would save real time;
- whether the README/demo make the product understandable in under 30 seconds;
- SmartScreen/Defender/install friction for the Windows installer.

## Known Limitations

- Windows is the primary tested platform.
- The installer may still trigger SmartScreen/Defender warnings for a young/unsigned app.
- Paste-to-previous-window depends on Windows foreground/focus behavior, target apps, timing, and paste shortcuts.
- Scripts are trusted local automation, not a secure sandbox or marketplace.
- AI is optional and disabled by default. Selected-content AI actions may send selected content to the configured provider.
- Rich clipboard formats are still evolving. Text and image-only capture exist; full HTML/RTF/custom-format fidelity is not a compatibility promise.
- Copicu is CopyQ-inspired, not CopyQ-compatible.

## Validation Before Publishing

Run from repo root:

```powershell
npm run build
npm run scripts:examples:test
npm run capabilities:drift:test
npm run ai:planner:test
cargo check --manifest-path src-tauri/Cargo.toml --tests
npm run rust:test
npm run tauri:build
```

For a full visual release gate, also run:

```powershell
npm run visual:check
```
