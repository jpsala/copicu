# Product

## Register

product

## Users

Power desktop users who copy, search, preview and reuse text, code, URLs, HTML and images throughout the day. They use Copicu as a local, private, keyboard-first clipboard tool while working across editors, browsers and native Windows apps.

## Product Purpose

Copicu is a local clipboard manager inspired by CopyQ, built with Tauri, TypeScript, Rust and SQLite. It should make captured clipboard history fast to search, safe to preview, easy to reuse, and extensible through personal scripts without becoming a CopyQ compatibility clone.

## Brand Personality

Fast, discreet, precise. The interface should feel like a native utility for focused work: compact, predictable and polished, with motion used only for state and focus.

## Anti-references

Do not make a marketing-style landing page, decorative SaaS dashboard, oversized hero, playful novelty UI, or full CopyQ clone. Avoid heavy visual effects, gratuitous transparency, fragile custom window behavior, and UI that hides core workflows behind ornamental chrome.

## Design Principles

- Keep the first screen useful: the picker is the product surface, not a shell around it.
- Optimize for keyboard flow first, then make pointer actions obvious and low-friction.
- Treat native behaviors as product risks: focus, shortcut, paste and window state must stay reliable.
- Use custom UI where it improves the quick picker, but keep long task surfaces native until validated.
- Preserve privacy: examples, tests and logs use synthetic data and never store real clipboard payloads.

## Accessibility & Inclusion

Aim for readable contrast, visible focus states, reduced-motion compatibility and controls that remain usable at narrow picker sizes. Standard affordances are preferred over invented interactions.
